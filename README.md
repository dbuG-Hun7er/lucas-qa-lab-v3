## Sobre o projeto

O **Lucas QA Lab** é meu portfólio pessoal e hub de projetos, criado para reunir minha trajetória profissional, meus estudos em Quality Assurance, meus projetos técnicos, minha experiência anterior com soldagem e qualidade industrial, além de ideias autorais como o **QA Lendário**, a **Taverna QA** e meu futuro guia de **Aion 2**.

A proposta deste site não é ser apenas um currículo online.  
Ele foi pensado como um espaço de identidade, evolução e construção pública.

Aqui eu apresento:

- minha transição da soldagem para a área de QA;
- minha base prática com qualidade, falhas, segurança e melhoria contínua;
- meus projetos com testes manuais, automação, API, performance, mobile e banco de dados;
- minha visão sobre cybersecurity como extensão da qualidade;
- meu projeto autoral **QA Lendário**;
- a ideia da **Taverna QA**, uma comunidade social tech com alma de RPG;
- meus estudos de MMORPG e sistemas de jogo através do **Aion 2**.

---

## Frase central

> **Da solda ao software, eu carrego a mesma missão: encontrar falhas, proteger sistemas e construir soluções melhores.**

Essa frase representa a conexão entre minha experiência no mundo físico e minha transição para o mundo digital.

Antes de testar software, eu já lidava com falhas reais, manutenção, protótipos, segurança operacional, retrabalho, melhoria contínua e responsabilidade com processos.  
Hoje aplico essa mesma mentalidade em testes de software, automação, API testing, cybersecurity e qualidade de sistemas digitais.

## Como executar o projeto localmente

### 1. Clone o repositório

```bash
git clone https://github.com/dbuG-Hun7er/lucas-qa-lab.git
```

### 2. Acesse a pasta do projeto

```bash
cd lucas-qa-lab
```

### 3. Abra o arquivo `index.html`

Você pode abrir o arquivo diretamente no navegador.

Ou, se estiver usando o VS Code, pode usar a extensão **Live Server**.

---

## Como iniciar o projeto com Git

Caso o projeto ainda não esteja versionado:

```bash
git init
git add .
git commit -m "primeira versão do portfólio Lucas QA Lab"
```

Depois, crie um repositório no GitHub e conecte:

```bash
git branch -M main
git remote add origin https://github.com/dbuG-Hun7er/lucas-qa-lab.git
git push -u origin main
```

---

## Como publicar no GitHub Pages

1. Acesse o repositório no GitHub.
2. Vá em **Settings**.
3. Clique em **Pages**.
4. Em **Build and deployment**, selecione:
   - Source: `Deploy from a branch`
   - Branch: `main`
   - Folder: `/root`
5. Salve.
6. Aguarde o GitHub gerar o link.

O link ficará parecido com:

```txt
https://dbug-hun7er.github.io/lucas-qa-lab/
```
