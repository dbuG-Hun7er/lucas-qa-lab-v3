Quality Assurance

QA profissional: fundamento antes de ferramenta.

Esta página organiza minha visão técnica de QA, conectando testes manuais, API, automação, performance, mobile, BDD e evidências.

Visão de QA

QA é leitura de risco. O trabalho não é apenas executar roteiro, mas entender produto, usuário, negócio, regra, fluxo, exceção e impacto.

Testes manuais

Cenários, casos de teste, massa de dados, pré-condições, evidências, bug report e testes exploratórios com intenção.

API Testing

Validação de contrato, status code, payload, autenticação, autorização, erro, dados inválidos e comportamento fora da tela.

Automação

Automatizar com propósito: proteger fluxo crítico, reduzir retrabalho e dar confiança para alteração de código.

BDD / Cucumber

Transformar regra de negócio em conversa clara: Given, When, Then precisa alinhar entendimento antes de virar automação.

Performance

Avaliar tempo de resposta, carga, ramp-up, duração, gargalo, erro e estabilidade com ferramentas como JMeter e k6.

Mobile

Fluxos mobile exigem atenção a estado do app, teclado, rede, permissões, telas menores, timing e comportamento em dispositivo real.

Evidências

Bug bom é bug compreensível: título claro, passos, ambiente, resultado esperado, resultado obtido e impacto.

Jornada QA Jr

Construir base com projetos reais, GitHub organizado, README forte, prática técnica e evolução contínua.

QA Lendário

O bug não respeita currículo bonito. Ele respeita investigação.

O QA Lendário é minha visão de qualidade: pensar antes de clicar, questionar antes de aceitar, documentar antes de discutir e proteger o produto antes do incidente.

Qualidade não começa no teste. Começa na forma como você enxerga o risco.
